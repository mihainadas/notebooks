Acest notebook implementeaz o versiune a perceptronului, aa cum a fost introdus n lucrarea din 1958 a lui Frank Rosenblatt, "The Perceptron: A Probabilistic Model for Information Storage and Organization in the Brain."

Vom utiliza concepte matematice de baz, minimiznd algebr liniar (vectri i matrici) acolo unde este posibil.

Introducere n Perceptron

Un perceptron este un model computaional simplu al unui neuron. Acesta formeaz fundaia reelelor neuronale i funcioneaz prin preluarea de intrri ponderate i aplicarea unei funcii de prag pentru a produce o ieire. Acest notebook arat cum pot fi folosii perceptronii pentru a clasifica date separabile liniar.

Obiectiv

Obiectivul este s antrenm un model cu dou ponderi, $w_1$ i $w_2$, corespunztoare coordonatelor $x$ i $y$ ale unui punct $(x, y)$, mpreun cu un bias $b$. Acest lucru se aliniaz cu formula pendinte-intercept a unei drepte, $y = mx + c$.

Modelul va aborda o sarcin simpl de clasificare pe un set de date separabil liniar bazat pe urmtoarea funcie:

$ f: \mathbb{N} \to \mathbb{N}, \quad f(x) = \begin{cases} x, & \text{dac } x \bmod 2 = 0, \\ 2x, & \text{dac } x \bmod 2 = 1. \end{cases} $

Set de Date

Vom genera un set de date folosind Biblioteca Standard Python. Setul de date const n puncte determinate de funcia de mai sus, clasificate n dou clase. Aceast configuraie ne permite s antrenm eficient clasificatorul nostru perceptron.

import random


def generate_dataset(num_items=20, start=0, stop=100):
    random.seed(42)
    dataset = []
    x1_values = set()
    while len(dataset) < num_items:
        x1 = random.randint(start, stop)
        if x1 in x1_values:
            continue
        x1_values.add(x1)
        x2 = x1 if x1 % 2 == 0 else 2 * x1
        y = (
            0 if x1 == x2 else 1
        )  # (x1, x2) este etichetat ca Clasa 0 dac x1 este par, i Clasa 1 altfel
        dataset.append((x1, x2, y))
    return dataset


dataset = generate_dataset()

# acum s mprim setul de date n seturi de antrenament i testare
train_ratio = 0.8
num_train = int(len(dataset) * train_ratio)
dataset_train, dataset_test = dataset[:num_train], dataset[num_train:]
print(f"Setul de antrenament (n={len(dataset_train)}): {dataset_train}")
print(f"Setul de testare (n={len(dataset_test)}): {dataset_test}")

Definirea unui Clasificator Liniar

Cu setul nostru de date pregtit, ne concentrm acum pe fundaia matematic care permite modelului nostru s clasifice intrrile $x_{1}$ i $x_{2}$ n clasele $0$ sau $1$. Matematic, aceasta poate fi reprezentat astfel:

$ c: \mathbb{R}^2 \to \{0,1\}, \quad c(x_{1}, x_{2}) = \begin{cases} 1, & \text{dac } (x_{1}, x_{2}) \in \text{Clasa 1}, \\ 0, & \text{dac } (x_{1}, x_{2}) \in \text{Clasa 2}. \end{cases} $

Aceast clasificare folosete reprezentarea algebric a unei linii ntr-un sistem de coordonate carteziene, descris de:

$ z(x) = w_{1}x_{1} + w_{2}x_{2} + b, $

unde:

  • $w_{1}$ i $w_{2}$ sunt ponderi care influeneaz panta liniei, dictnd unghiul acesteia n raport cu axa $x$.
  • $b$ este bias-ul (sau interceptul), indicnd locul unde linia intersecteaz axa $y$.

Din graficul desenat mai sus, cele dou clase sunt vizibil separabile liniar, justificnd utilizarea unei frontiere de decizie liniare. Antrenarea ponderilor $w_{1}$ i $w_{2}$ folosind algoritmul Perceptron ofer o abordare ideal pentru stabilirea acestei frontiere.

Pentru a ilustra, considerai o linie definit de $w_{1}=1$, $w_{2}=0.5$, i $b=0$. Apariia acestei linii pe graficul anterior ajut la definirea modului n care separ bine clasele noastre.

In[14]:
zx = lambda x1, x2, w1, w2, c: w1 * x1 + w2 * x2 + c

def plot_zx(w1, w2, c):
    x2 = lambda x1: (
        (-w1 * x1 - c) / w2 if w2 != 0 else -c / w1 if w1 != 0 else c
    )  # acesta este pentru c ecuaia dreptei este w1*x1 + w2*x2 + c = 0, prin urmare x2 = (-w1*x1 - c) / w2
    x1_values = range(0, 101)
    x2_values = [x2(x1) for x1 in x1_values]
    plt.plot(x1_values, x2_values, label=f"{w1}x1+{w2}x2+{c}=0")
    plt.legend(loc="best")

def plot_datasets_and_zx(w1, w2, c):
    plot_datasets(dataset_train, dataset_test)
    plot_zx(w1, w2, c)

Evaluarea Performanei Clasificatorului

Cu $z = w_{1}x_{1} + w_{2}x_{2} + c$ definit ca fiind limita de decizie a clasificatorului nostru, am observat vizual c poate clasifica folosind anumite valori alese. Acum, s determinm computaional performana sa folosind metrica de acuratee.

Definirea Clasificatorului

nainte de a evalua performana clasificatorului, s l definim clar:

$ c: \mathbb{N} \times \mathbb{N} \to \{0,1\}, \quad c(x_{1},x_{2}) = \begin{cases} 1, & \text{dac } z(x_{1}, x_{2}) \geq 0, \\ 0, & \text{dac } z(x_{1}, x_{2}) < 0. \end{cases} $

n esen, aceasta nseamn c dac un punct $(x_1, x_2)$ se afl pe sau deasupra limitei de decizie $z(x_1, x_2)$, este clasificat ca $1$ (Clasa 1); altfel, este clasificat ca $0$ (Clasa 0).

S implementm clasificatorul n cod, i apoi s discutm cum putem evalua performana lui n mod eficient.

In[16]:
cx = lambda x1, x2, w1, w2, c: 1 if zx(x1, x2, w1, w2, c) >= 0 else 0


def accuracy(dataset, w1, w2, c):
    print(f"Calcularea acurateei pe setul de antrenament folosind w1={w1}, w2={w2}, c={c}")
    correct = 0
    for x1, x2, y in dataset:
        if y == cx(x1, x2, w1, w2, c):
            correct += 1
    print(
        f"Acurateea rezultat: {correct}/{len(dataset)}, sau {correct/len(dataset)*100:.2f}%"
    )
    return correct / len(dataset)


# Aplicarea funciei de acuratee pe setul de antrenament folosind cele dou seturi de greuti i bias, aa cum este artat mai sus, n primul exemplu
accuracy(dataset_train, -1.5, 1.1, -10)

# Aplicarea funciei de acuratee pe setul de antrenament folosind dou seturi de greuti i bias, aa cum este artat mai sus, n al doilea exemplu
accuracy(dataset_train, 0.1, 0.1, 0.5)

Discuie despre Acuratee

Anterior, am observat cum modificarea greutilor i a bias-ului afecteaz acurateea modelului nostru de perceptron. Aceast variaie apare deoarece fiecare set de greuti i bias definete o limit de decizie unic, care influeneaz capacitatea modelului de a clasifica corect punctele de date. Obiectivul principal este de a determina valorile "optime" pentru aceti parametri, un proces realizat prin antrenarea modelului.

Antrenarea Modelului

Pe baza analizei noastre, vom antrena modelul nostru de perceptron iterativ, ajustnd fin greutile i bias-ul. Acest antrenament implic ajustri pe parcursul mai multor epocicicluri de nvare prin setul de datepermind modelului s mbunteasc treptat acurateea. Ideea este de a actualiza iterativ aceti parametri pentru a se potrivi mai bine sarcinii noastre de clasificare i caracteristicilor setului de date, mbuntind astfel performana general a modelului.

# Mai nti, s iniializm ponderile i biasul la zero
w1, w2, c = 0, 0, 0

# Acum s definim rata de nvare
learning_rate = 0.1

# Acum s definim numrul de epoci
num_epochs = 5

# Crem un DataFrame pentru a stoca detaliile epocilor
epoch_details = pd.DataFrame(columns=["epoch", "x1", "x2", "y", "z", "y_hat", "w1", "w2", "c", "err"])

# S ncepem acum bucla de antrenare
for epoch in range(num_epochs):
    print(f"Epoca {epoch+1}")
    error = 0
    for x1, x2, y in dataset_train:
        z = zx(x1, x2, w1, w2, c)
        y_hat = 1 if z >= 0 else 0
        if y != y_hat:
            error += 1 # Numrm numrul de erori
        w1 += learning_rate * (y - y_hat) * x1
        w2 += learning_rate * (y - y_hat) * x2
        c += learning_rate * (y - y_hat)
        print(f"  x1={x1}, x2={x2}, y={y}, z={z:.2f}, y_hat={y_hat}, w1={w1:.2f}, w2={w2:.2f}, c={c:.2f}")
        # Adugm detaliile n DataFrame
    epoch_details = pd.concat([epoch_details, pd.DataFrame([{
        "epoch": epoch + 1,
        "x1": x1,
        "x2": x2,
        "y": y,
        "z": z,
        "y_hat": y_hat,
        "w1": w1,
        "w2": w2,
        "c": c,
        "err": error
    }])])

Limitri i Lucrri Viitoare

Dei perceptronul este eficient pentru date care pot fi separate liniar, ntmpin dificulti cu seturile de date care nu pot fi separate printr-o singur linie dreapt. Pentru date care nu pot fi separate liniar, sunt necesare modele mai complexe. ncercrile viitoare ar putea include explorarea:

  • Neuronul Liniar Adaptativ (Adaline), folosind activri liniare continue.
  • Modele de regresie logistic folosind activri sigmoid.
  • Perceptroni multi-strat (MLPs) pentru seturi de date mai complexe.
  • Reele Neuronale Convoluionale (CNNs) pentru date de imagine.
  • Reele Neuronale Recurente (RNNs) pentru date de secven.
  • Reele Generative Adversariale (GANs) pentru crearea de mostre de date.
  • Modele Transformer pentru diverse sarcini de procesare a limbajului natural, incluznd configuraii numai cu encoder, numai cu decoder i encoder-decoder.

Aceste proiecte promit nvare i explorare ulterioar, aa c rmnei pe aproape pentru viitoarele progrese!
